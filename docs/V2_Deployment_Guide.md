# V2 Web Application Deployment Guide

This document defines the software requirements and installation steps for the **Evaluation System V2**.

There are 3 environments involved in this system.

## 1. Environment Checklist

### A. Developer PC (Machine used to write code)
*   **Purpose**: Coding, Compiling React to HTML/JS.
*   **Requirements**:
    *   [x] **Node.js** (v18+) & **npm**: To build the React frontend.
    *   [x] **Python 3.9+**: To test the backend locally.
    *   [x] **VS Code**: Editor.

### B. Target Device (The "Destination" / ARM Board / Zybo)
*   **Purpose**: Running the actual system, controlling hardware, serving the web page.
*   **Requirements** (Must be installed on the Linux OS):
    *   [x] **Python 3.9+**: Core runtime.
    *   [x] **pip**: To install dependencies.
    *   [x] **FastAPI & Uvicorn**: (`pip install fastapi uvicorn`) The web server software.
    *   [x] **Systemd**: (Built-in to Linux) To auto-start the app on boot.
    *   [ ] **NO Node.js required**: We only copy the *built* files here.
    *   [ ] **NO Internet required**: Can run on local LAN.

### C. Client Machine (The User / End-Device)
*   **Purpose**: Operator controls the system.
*   **Requirements**:
    *   [x] **Web Browser** (Chrome, Edge, Firefox, Safari).
    *   [ ] **NO Installation**: The user installs *nothing*. They just type the IP address (e.g., `http://192.168.1.105`).

---

## 2. Installation Steps (For the Target ARM Board)

### Step 1: Install Python Dependencies
Open the terminal on the ARM Board (SSH) and run:
```bash
sudo apt-get update
sudo apt-get install python3 python3-pip
pip3 install fastapi uvicorn pydantic requests
```

### Step 2: Transfer Files
Copy your project files from the Dev PC to the ARM Board (e.g., via `scp` or USB stick).
*   **Location**: `/opt/eval_system/`
*   **Files Needed**:
    *   `/frontend` (The `dist` folder generated by `npm run build`)
    *   `main.py` (The Python FastAPI server)
    *   `hardware_driver.py` (Your logic for VCD/GPIO)

### Step 3: Create Startup Service
To make the web app start automatically when the board turns on:

1.  Create file `/etc/systemd/system/eval_webapp.service`:
    ```ini
    [Unit]
    Description=Eval System V2 Web Interface
    After=network.target

    [Service]
    User=root
    WorkingDirectory=/opt/eval_system
    ExecStart=/usr/bin/python3 -m uvicorn main:app --host 0.0.0.0 --port 80
    Restart=always

    [Install]
    WantedBy=multi-user.target
    ```
2.  Enable and Start:
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable eval_webapp
    sudo systemctl start eval_webapp
    ```

## 3. Usage
Simply open `http://<BOARD_IP_ADDRESS>` on any computer connected to the same network.
